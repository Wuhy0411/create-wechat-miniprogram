# 微信小程序从零到发布：新手完整指南

> 调用本 skill 后，你会得到一份**按步骤执行**的新手指南。从注册账号到发布上线，每一步都写清楚了。

---

## 适用场景

- 你第一次创建微信小程序，不知道从哪里开始
- 你手上有一份小程序项目代码，想把它跑起来并发布
- 你需要在微信开发者工具中配置云开发（数据库 + 云函数）

---

## 前置条件检查

在开始之前，确认你已有：
- 一个未被注册过微信公众平台的邮箱（建议 QQ 邮箱或 Gmail）
- 一个绑定银行卡的微信账号（用于扫码认证）
- 真实姓名 + 身份证号（个人主体注册需要）

---

## 完整流程（8 步）

### 第一步：注册小程序账号

1. 浏览器打开 **https://mp.weixin.qq.com/** → 右上角「立即注册」→ 选择「小程序」
2. **邮箱激活**：填写邮箱 → 设置密码 → 去邮箱点激活链接
3. **选择主体类型**：个人开发者选「个人」（免费），企业选「企业」
4. **实名认证**：填写真实姓名 + 身份证号
5. **扫码验证**：用绑了银行卡的微信扫码，该微信号成为管理员
6. **设置小程序信息**：填写名称（**一旦设置不能随意修改**）、头像、简介
7. **获取 AppID**：登录后台 → 左侧「开发」→「开发管理」→「开发设置」→ 复制 AppID（以 `wx` 开头的一串字符）

> ⏱ 约 10 分钟

---

### 第二步：开通微信云开发

1. 登录 https://mp.weixin.qq.com/
2. 左侧菜单 →「开发」→「云开发」→ 点「开通」
3. 填写环境名称（如 `my-app`、`pro`）
4. 计费模式选「按量计费」（个人使用免费额度完全够用）

**免费额度参考**：
| 资源 | 免费额度/月 |
|------|-----------|
| 数据库读 | 20 万次 |
| 数据库写 | 10 万次 |
| 云存储 | 2 GB |
| 云函数调用 | 100 万次 |

> ⏱ 约 2 分钟

---

### 第三步：下载安装微信开发者工具

这是开发微信小程序的**唯一必需工具**，集成了代码编辑、模拟器、调试、云开发管理、上传发布等功能。

1. 下载地址：**https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html**
2. 选择「稳定版」下载
3. 双击安装包，一路「下一步」
4. 启动后用**管理员微信号**扫码登录

> ⏱ 约 5 分钟

---

### 第四步：导入项目代码

1. **配置 AppID**：打开项目文件夹中的 `project.config.json`，确认 `appid` 字段已填入你的 AppID（替换 `__YOUR_WECHAT_APPID__`）
2. 打开微信开发者工具 → 点「导入项目」（或左上角「+」号）
3. 选择项目目录（如 `my-miniprogram` 文件夹）
4. AppID 会自动从 `project.config.json` 读取
5. 点「导入」

> 首次导入如果弹框提示「云开发环境未关联」，先点确定忽略，下一步配置。

---

### 第五步：配置云开发（创建数据库 + 上传云函数）

**这是最关键的步骤。**

#### 5.1 打开云开发控制台
左侧工具栏 → ☁️「云开发」图标

#### 5.2 创建数据库集合
1. 点「数据库」→「新建集合」
2. 集合名称填项目需要的名称（如 `dishes`）
3. 点「确定」

#### 5.3 设置数据库权限
1. 点刚创建的集合 →「权限设置」
2. 选择「自定义安全规则」，替换为：
```json
{ "read": true, "write": true }
```
3. 保存

#### 5.4 添加索引（推荐）
点「索引管理」→「添加索引」：
- 索引1：字段 `id`，升序，勾选「唯一」
- 索引2：字段 `category`，升序，不勾唯一

#### 5.5 上传云函数
1. 在左侧文件树中找到 `cloudfunctions` 文件夹
2. 右键子文件夹（如 `init-db`）
3. 选择「上传并部署：云端安装依赖」
4. 等待底部状态栏显示「上传成功」

#### 5.6 运行云函数填充数据
1. 云开发控制台 →「云函数」
2. 找到刚上传的云函数 → 点右侧「测试」
3. 直接点「开始测试」（不需要改参数）
4. 返回结果应显示类似 `{"code":0,"message":"初始化成功","total":364}`

#### 5.7 验证数据
云开发 → 数据库 → 集合 →「管理」，确认能看到数据记录。

> ⏱ 约 5 分钟

---

### 第六步：本地预览测试

1. 点顶部「编译」按钮（或 `Ctrl+B`）
2. 左侧模拟器中查看小程序界面
3. 测试基本功能：浏览页面、切换分类、进入详情、返回首页
4. 如果页面空白：检查云函数是否运行成功、数据库是否有数据

> ⏱ 约 10 分钟

---

### 第七步：真机扫码预览

1. 点顶部「预览」按钮 → 生成二维码
2. 用手机微信扫码 → 手机端体验小程序
3. 在手机上测试所有功能（浏览、详情、编辑、上传图片等）

> ⚠️ 只有扫码的人能打开，其他人打不开（还没发布）

---

### 第八步：发布上线

#### 8.1 上传代码
1. 微信开发者工具中点右上角「上传」
2. 版本号填 `1.0.0`（后续更新递增）
3. 项目备注写「首次发布」
4. 点「上传」

#### 8.2 提交审核
1. 浏览器登录 https://mp.weixin.qq.com/
2. 左侧「管理」→「版本管理」
3. 在「开发版本」列表中找到刚上传的版本
4. 点「提交审核」
5. 填写功能备注 + 选择类目（建议「工具 > 效率」）
6. 勾选「用户隐私保护指引」确认 → 提交

#### 8.3 等待审核
- 审核时间通常 **1-3 个工作日**
- 审核结果通过「服务通知」发给管理员微信
- 可在「版本管理」页面查看状态

#### 8.4 发布
审核通过后 → 回到「版本管理」→ 点「发布」

**发布后任何人搜索小程序名称都能找到并使用。**

#### 8.5 获取小程序码
后台「设置」→「基本设置」→ 下载「小程序码」图片，可分享给朋友扫码打开。

> ⏱ 审核 1-3 天 | 💰 免费

## 常见问题速查

### Q1：上传云函数报 npm 错误
**原因**：Windows PowerShell 默认禁止运行脚本

**解决**：
1. 搜索框输入 `PowerShell` → 右键「以管理员身份运行」
2. 输入：`Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned`
3. 输入 `Y` 回车确认
4. 回到微信开发者工具重新上传

### Q2：编译后页面空白
- 检查云函数是否运行成功
- 检查云开发 → 数据库 → 集合里是否有数据
- 检查集合名称是否和代码中一致

### Q3：编辑保存提示"保存失败"
- 检查数据库集合权限是否为 `{"read": true, "write": true}`
- 如果权限是「仅创建者可写」，改成自定义规则

### Q4：小程序审核不通过
常见原因：
- **类目选择不对**：重新选「工具 > 效率」
- **页面功能不完整**：确保所有页面能正常打开
- **缺少隐私说明**：在「设置」→「基本设置」中补充「用户隐私保护指引」
- **个人主体做需资质的功能**：个人小程序不能做外卖、支付、医疗等

### Q5：换电脑了怎么继续开发？
1. 把项目文件夹拷到新电脑
2. 装微信开发者工具，扫码登录
3. 点「导入项目」→ 选择项目文件夹
4. AppID 自动读取，云数据在微信云端不需要迁移

### Q6：云开发收费吗？
按量计费，个人使用通常**不收费**。超出免费额度后按量计费（约 0.1 元/万次）。

### Q7：以后怎么更新版本？
1. 在微信开发者工具中修改代码
2. 点「编译」测试
3. 点「上传」，版本号递增（如 `1.0.1`）
4. 到小程序后台提交审核
5. 审核通过后发布

## 关键名词解释

| 名词 | 说明 |
|------|------|
| **AppID** | 小程序唯一 ID，配置项目时需要 |
| **云开发** | 微信提供的后端服务（数据库 + 云存储 + 云函数），不需要自己买服务器 |
| **云函数** | 运行在微信服务器上的 Node.js 代码 |
| **云存储** | 存图片、视频等文件 |
| **数据库** | 存结构化数据（类似 Excel 表格） |
| **审核** | 微信对小程序功能 + 内容的审查 |
| **版本管理** | 管理小程序从开发 → 审核 → 发布的入口 |
